# 本次上架前修复与优化摘要

最后更新：2026-03-23

## 已落地的核心修复
- 客户阶段支持“手动维护 / 自动推断”双模式，不再被订单状态无条件覆盖。
- 已收金额、待回款、付款状态统一走付款流水口径。
- 工作室资料统一回写到设置与确认单分享内容。
- 跟进提醒开关已持久化，关闭后不会再次被系统自动排程。
- 自动创建拍前确认跟进只会更新系统来源的那一条记录，不会误覆盖其他跟进。
- 删除存在历史订单/跟进的客户时，系统会自动改为归档以保留经营历史。
- 不同 Apple ID 登录会切换到隔离工作区，避免串数据。
- 已新增“删除账号与当前工作区”入口，便于满足送审要求。

## 已完成的送审收口
- 移除未启用的位置/天气服务文件与相关工程残留。
- 移除未使用的 `BusinessModulesSupport.swift`。
- 将“系统日历 / Google Calendar 双向同步”调整为“外部日历整备”，避免 UI 宣传与实际能力不一致。
- 将“多人实时协作”调整为“团队权限与操作留痕”，保留真实可用的本地权限与留痕能力。
- 限制为竖屏，减少未充分适配的横屏审核风险。
- 补充并更新 App Store Listing、Review Notes、Support、Privacy 文档模板。

## 已完成的性能与稳定性优化
- 为付款流水增加按订单缓存，减少列表、详情和报表中的重复遍历。
- 为客户订单、客户跟进增加关系缓存，减少客户页统计开销。
- 经营分析改为缓存快照，不再在多个页面实时重复全量计算。
- 完整备份升级为“工作区 JSON + Attachments 目录”的包结构。
- 恢复时支持读取完整备份包，并同步恢复附件文件。
- Google Calendar 敏感字段不再进入普通快照/备份持久化。
- 附件导入增加 security-scoped 访问与更准确的 MIME 识别。
- 清理 `.derived`、`.codex-home`、`__MACOSX`、`xcuserdata` 等无用交付内容。

## 当前版本的真实能力边界
- 外部日历：当前正式版只展示接入说明与状态记录，不默认启用系统日历或 Google Calendar 自动同步。
- 团队能力：当前正式版提供本地角色权限、导出控制和关键操作留痕，不提供真正的实时多人协作。
- 定位/天气：当前正式版不启用。

## 建议你在 Mac/Xcode 上做的最终确认
- Clean Build Folder
- 真机/模拟器回归：客户、订单、跟进、回款、确认单分享、完整备份恢复
- App Store Connect 中填写已部署的 Privacy Policy URL / Support URL
- 真机检查竖屏布局与文件分享行为
