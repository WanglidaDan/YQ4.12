# 经营中心扩展能力现状说明

最后更新：2026-03-23

## 已接入的业务模块
### 1. 合同 / 报价 / 收据 / 发票
- 已在数据层和经营中心入口中接入业务单据能力。
- 可按客户/订单管理文档记录、内容项与分享文本。
- 当前版本更偏向结构化业务记录与分享，不伪装为正式签章平台。

### 2. 外部日历整备
- 已有订单级状态记录、接入说明与权限入口位置。
- 当前正式版不默认启用系统日历写入/回写，也不默认连接 Google Calendar。
- 相关界面仅用于展示准备状态，避免送审时出现“宣传已支持但实际未接通”的问题。

### 3. 附件与参考资料
- 支持导入本地文件、保存外部链接、按客户/订单管理资料。
- 附件会复制到 App 本地资料库，并纳入完整备份恢复链路。

### 4. 团队权限与操作留痕
- 提供 Owner / Admin / Producer / Photographer / Finance / Viewer 等角色。
- 当前正式版提供本地权限控制、导出控制与关键操作留痕。
- 不对外宣称实时多人协作，避免与当前底层能力不匹配。

### 5. 经营分析报表
- 已接入签约额、回款、待收、客单价、渠道表现等经营分析数据。
- 当前版本通过缓存快照减少重复计算，优先保障工作台与经营中心流畅度。

## 本次为送审额外做的收口
- 统一经营中心文案，改为“外部日历整备 / 团队权限与操作留痕”。
- 去掉未真正生效的实时协作与自动日历同步宣传。
- 完整备份升级为包含附件的备份包。
- 敏感的 Google Calendar 凭证信息不再写入普通快照与导出内容。

## 本地最终验证建议
- 真机确认经营中心文案、入口和权限逻辑正常。
- 验证附件导入、分享、完整备份和恢复。
- 验证游客模式、Apple 登录、iCloud 同步开关、删除账号与工作区流程。
